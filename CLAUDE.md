# User instructions (loaded in every project)

## Git

- **Never `git push` (any form, including force pushes and variants like `git -C <dir> push`) without explicit per-action confirmation from me.** Approval for one push does NOT carry over to the next one. Committing locally is fine; publishing is not, unless I say so for that specific push. A `Bash(git push:*)` ask-rule also enforces this in `~/.claude/settings.json`, but honor the intent even where the pattern wouldn't match.
